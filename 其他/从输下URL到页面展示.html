一：请求阶段（从客户端向服务器发送请求<也叫做request请求阶段>）： 1：（URL地址解析）输入地址：从书签、历史记录等地方给你智能补全路径让你选择，更有甚者如chrome，直接从缓存拉出页面展示； 2：（DNS域名解析）浏览器查找域名的IP地址：一般先会在本地host文件查找有没有相对于的规则，有就直接用host里面的IP地址， 如果没有，浏览器就会发一个DNS请求到本地DNS服务器（一般是网络接入服务器商提供的， 如中国电信）， 请求到达本地的DNS服务器后，首先查询它的缓存记录， 有救直接返回结果了， 没有本地DNS服务器会向根DNS服务器进行请求（递归的过程）。 倘若根DNS服务器还是没有，就会告诉本地DNS服务器，这个时候你就可以到域服务器上去进行查询了，并给出域服务器的地址（迭代的过程）， 本地DNS服务器想域服务器发送请求后，会得到一个你的域名的解析服务器的地址，最后，本地DNS服务器想域解析服务器发送请求，得到域名和IP的 对应的关系，本地DNS服务器会把IP返回电脑并缓存下来以便下一次使用。 DNS：域名系统，类似电话簿的功能，我们习惯记住网站名称（www.baidu.com），计算机记住IP地址，NDS就对应起来 我们输入网址 DNS就去找到IP地址； 3：（和服务器建立TCP链接）TCP建立连接（三次握手） 4：发送HTTP请求 二：响应阶段 ； 三：断开链接 四：渲染页面 详情请看： https://juejin.im/post/6844904162166063118 window.onload 和 window.DOMContentLoaded 区别 window.addEventListener('load',function(){ // 页面资源全部加载完毕 }) window.addEventListener('DOMContentLoaded',function(){ // Dom 渲染完就执行，图片 视频没有加载完 })
